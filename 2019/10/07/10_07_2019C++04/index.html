C++ Primer第五版 6.33给出了类似的东西，我们对比来看看:

C++Primer是这样解释这个定义的:

  1. func(int i) 表示调用func函数时需要一个int类型实参。
  2. (*func(int i)) 意味着我们可以对函数调用的结果执行解引用操作。
  3. (*func(int i))[10] 表示解引用func的调用将得到一个大小为10的数组。
  4. int (*func(int i))[10] 数组中元素类型为int。

所以我们可以知道:

  1. __countof_helper(_UNALIGNED _CountofType(&_Array)[_SizeOfArray]) 
    调用__countof_helper函数需要里面那个乱七八糟的东西。
  2. (*__countof_helper(_UNALIGNED _CountofType(&_Array)[_SizeOfArray]))
    函数调用的结果解引用。
  3. (*__countof_helper(_UNALIGNED _CountofType(&_Array)[_SizeOfArray]))[_SizeOfArray]
    解引用函数调用将得到一个大小为10的数组。
  4. char (*__countof_helper(_UNALIGNED _CountofType(&_Array)[_SizeOfArray]))[_SizeOfArray]
    数组元素类型为char。

原博中需要注意的是array_t的类型,我在这里迷糊了很久，后面给出的C++Primer中的解释更好理解。
